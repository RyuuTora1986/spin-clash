# Gameplay UX Visual Optimization Taskboard

## Purpose

This taskboard breaks the approved optimization design into execution units that can be implemented and reviewed independently.

Each task is small enough to reason about, but still tied to a real UX outcome.

## Wave 1: Comprehension And Momentum

### GX-001 Battle Opening Guidance

- [ ] Capture current battle-opening screenshots and note the current instructional weak points
- [ ] Define the minimum opening instruction set for first-session comprehension
- [ ] Design the pre-launch micro-brief layer
- [ ] Design the near-control reinforcement layer
- [ ] Review the opening state for clarity versus visual noise
- [ ] Re-run battle start through the real player path and confirm the first action is easier to infer

### GX-002 Round Result Readability

- [ ] Capture current round-result state with background competition visible
- [ ] Separate current result information into headline / cause / next step
- [ ] Redesign the visual dominance of the result card over the battle background
- [ ] Rewrite cause summary into player-facing language if needed
- [ ] Review whether the result card now teaches instead of merely reports
- [ ] Re-run a losing round and confirm the player can explain what happened

### GX-006 Match Result CTA Hierarchy

- [ ] Capture current loss-state and win-state result surfaces
- [ ] Classify all CTAs into primary / optional / utility
- [ ] Define the intended next action for loss states
- [ ] Define the intended next action for win states
- [ ] Rebuild button hierarchy around action priority rather than visual parity
- [ ] Re-run the result flow and confirm the player can choose the likely next action with less hesitation

## Wave 2: Decision Quality

### GX-003 Home Entry Hierarchy

- [ ] Capture current home first-scan path
- [ ] Measure how much visual weight the showcase consumes relative to mode entry
- [ ] Define the desired distinction between Champion Path and Quick Battle
- [ ] Recompose the home hierarchy so the play-start question resolves faster
- [ ] Review whether identity was preserved while entry clarity improved

### GX-004 Champion Path Decision Support

- [ ] Capture the current path screen with current node, reward, and surrounding metadata visible
- [ ] Mark which information should be primary, secondary, and supporting
- [ ] Recompose the path around the current actionable node
- [ ] Increase reward promise clarity without reintroducing clutter
- [ ] Review whether a player can identify challenge and reward in one fast glance

### GX-005 Workshop Decision Support

- [ ] Capture current workshop cards with scrap balance visible
- [ ] Identify where comparison currently slows down
- [ ] Strengthen current-versus-next upgrade readability
- [ ] Improve affordability and immediate effect readability
- [ ] Review whether the page now supports choosing rather than merely reading

## Wave 3: Trust And Reading Surfaces

### GX-007 Desktop Public Info Shell

- [ ] Capture current desktop public info shell
- [ ] Identify what makes it feel like a popup rather than a page
- [ ] Define the desktop reading-shell layout model
- [ ] Review whether the page now feels like a native product surface

### GX-008 Multilingual Mobile Info Header

- [ ] Capture current mobile info headers in English, Chinese, and Japanese
- [ ] Identify where header logic breaks across locales
- [ ] Define one stable mobile header pattern for all languages
- [ ] Review whether title, back action, and body start now share one reading rhythm

## Review Rule For Every Task

No task is complete until these three are recorded:

- [ ] What player question became easier to answer?
- [ ] What structural issue was solved?
- [ ] What potential drift was checked and rejected?
